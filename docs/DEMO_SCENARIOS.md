# Aivo-v2 Demo演示场景设计

## 场景1：日常闲聊
**目标**: 展示自然对话和情感表达能力

```
用户: "你好，我是小明"
Aivo: "你好小明！很高兴认识你" [情感: happy, 动作: wave]

用户: "今天天气真好"
Aivo: "是啊，好天气总是让人心情愉快呢" [情感: happy, 动作: nod]

用户: "我有点累了"
Aivo: "那休息一下吧，别太辛苦自己" [情感: sad, 动作: none]
```

**亮点**:
- 上下文记忆（记住名字）
- 情感自动识别
- 动作自然配合

## 演示流程建议

### 答辩演示顺序（10分钟）

1. **开场（1分钟）**
   - 打开Web界面
   - 介绍系统架构（全MiMo API，0显存）

2. **基础对话（2分钟）**

   - 展示表情切换

6. **总结（1分钟）**
   - 查看统计数据
   - 技术亮点回顾

## 答辩问题预案

### Q1: 为什么不用开源大模型？
**A**:
- H20显存宝贵，留给数字人驱动
- MiMo工具调用稳定，JSON输出准确
- 开发效率高，专注智能体逻辑

### Q2: 如何保证实时性？
**A**:
- 全API化，网络延迟<500ms
- ASR+LLM+TTS并行优化
- 实测端到端<3秒

### Q3: 如何扩展新工具？
**A**:
- 基于BaseTool继承
- 自动注册到tool_registry
- LLM自动识别和调用

### Q4: 数据隐私如何保证？
**A**:
- 对话历史仅本地存储
- 可选本地部署方案
- API传输加密

### Q5: 成本如何控制？
**A**:
- 约¥1.5/小时
- 课设demo总成本<¥10
- 可切换到免费开源方案

## 展示技巧

1. **准备好的测试对话**：提前在文本文件中准备好问题，复制粘贴避免打字错误
2. **开两个窗口**：一个浏览器+一个终端（显示后台日志）
3. **准备备用方案**：如果网络不好，准备录屏视频
4. **数据可视化**：演示前清空统计，演示中实时显示数字增长
5. **对比演示**：先演示简单对话，再演示复杂工具调用，形成对比
