# 架构方案对比（全API化后）

## 方案A：完全云端化（推荐，0显存）

### 技术栈
| 模块 | 方案 | 成本 |
|------|------|------|
| ASR | MiMo API | ¥0.006/分钟 |
| LLM大脑 | MiMo mimo-v2.5-pro | 按tokens计费 |
| TTS | MiMo API | ¥0.02/分钟 |
| 数字人视频 | **商业API** | 见下表 |

### 数字人API选择

| 服务 | 价格 | 特点 | 适合场景 |
|------|------|------|----------|
| [D-ID](https://www.d-id.com/) | $0.12/视频秒 | 效果最好，支持情感 | 对效果要求高 |
| [HeyGen](https://www.heygen.com/) | $0.12/分钟 | 多语言，音色丰富 | 商业级项目 |
| [腾讯智影](https://zenvideo.qq.com/) | ¥0.1-0.3/分钟 | 国产，审核宽松 | 课设demo |
| [百度智能云](https://cloud.baidu.com/product/speech) | ¥0.15/分钟 | 有免费额度 | 预算有限 |

### 架构图
```
Mac 客户端（全部在本地完成）
├── 录音
├── MiMo ASR → 文本
├── MiMo LLM → {speech, emotion, action}
├── MiMo TTS → 音频
└── 数字人API → 视频播放

✅ 优点：
- 0显存占用，不需要H20 DSW
- 部署简单，Mac上运行即可
- 成本可控（约¥2-5/小时）

❌ 缺点：
- 数字人API有延迟（3-10秒）
- 需要额外付费
- 对网络依赖高
```

---

## 方案B：DSW部署轻量级数字人（需10-15GB显存）

### 轻量级方案
使用**2D Live2D**替代3D渲染：
- [Live2D Cubism](https://www.live2d.com/)：2D动画，显存需求小
- [VTube Studio](https://denchisoft.com/)：虚拟主播方案
- **自实现**：静态人物图片 + 口型动画 + 表情切换

### 架构图
```
Mac 客户端                  H20 DSW
├── 录音                    ├── FastAPI后端
├── → DSW                   ├── Live2D引擎（轻量）
                           └── 视频流推送

显存需求：~10GB（可接受）
```

---

## 推荐决策

### 课设Demo场景（推荐方案A）

**理由：**
1. 课设周期短，用API快速落地
2. Demo总时长<1小时，成本可控（<¥5）
3. 展示重点在"智能体工作流规划"，不是渲染技术
4. 商业API效果更好，答辩演示更稳定

### 长期项目/产品化（考虑方案B）

**理由：**
1. 成本可控，避免API持续付费
2. 可自定义数字人形象
3. 延迟更低，用户体验好

## 立即行动建议

既然你已经配好`.env`且都用MiMo API，我建议：

1. **先在Mac本地测试完整链路**（录音→ASR→LLM→TTS→播放音频）
2. **数字人暂时用静态图片 + 音频播放**（最简单）
3. **等核心功能跑通后**，再决定：
   - 要效果好 → 接入数字人API（D-ID/腾讯智影）
   - 要省钱/可控 → DSW部署Live2D轻量方案

这样可以先把"智能体能力"（工具调用、工作流规划）展示出来，数字人只是锦上添花。
