## 💡 答辩问题预案

### Q1: 为什么不用开源大模型？
**A**:
显存预算优化。H20有96GB显存，但我们选择把显存留给「没有API的模块」，比如数字人视频渲染。

LLM用API的优势：
1. 工具调用稳定（MiMo原生支持Function Calling）
2. JSON输出准确（开源小模型容易翻车）
3. 0显存占用
4. 开发效率高

如果未来有需求，可以切换到开源模型，架构支持。

---

### Q2: 如何保证实时性？延迟多少？
**A**:
我们进行了完整的性能测试：

实测数据：
- LLM响应：1-3秒
- TTS合成：<2秒
- 端到端延迟：2-5秒

优化手段：
1. 全API调用，网络延迟<500ms
2. 保留最近10轮对话，减少token
3. 异步处理（可选）

目标<5秒，实测达标 ✓

---

### Q3: 如何扩展新工具？
**A**:
采用插件式架构，非常简单：

```python
# 1. 继承BaseTool
class NewTool(BaseTool):
    @property
    def name(self): return "tool_name"

    @property
    def description(self): return "工具说明"

    def execute(self, **kwargs):
        # 工具逻辑
        return result

# 2. 注册
tool_registry.register(NewTool())
```

LLM会自动识别和调用，无需修改核心代码。

### Q8: 如果网络断了怎么办？
**A**:
当前方案依赖网络（API调用）

离线方案：
1. 部署开源模型到DSW
   - Qwen本地版
   - Whisper本地ASR
   - 需要40-50GB显存

2. 混合方案
   - LLM离线
   - ASR/TTS用API

权衡：
- 课设demo：网络方案（快速）
- 生产环境：离线方案（稳定）
