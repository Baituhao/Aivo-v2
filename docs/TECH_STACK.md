# 技术栈最终确认

## 音视频处理

| 模块 | 方案 | 说明 |
|------|------|------|
| ASR（语音识别） | **小米MiMo ASR** | 替代Whisper，云端API，0显存占用 |
| TTS（语音合成） | **小米MiMo TTS** | 8种预设音色+情感控制，云端API |
| 数字人驱动 | **Linly-Talker** | 集成MuseTalk/SadTalker，H20 DSW部署 |

## 智能体核心

| 模块 | 方案 | 说明 |
|------|------|------|
| 对话大脑 | **Qwen（通义千问）** | 工具调用稳定，中文优秀，有免费额度 |
| 工作流编排 | **Dify** | 可视化编排，支持工具调用 |
| 动作协议 | **自研JSON协议** | 控制表情+动作 |

## 完整数据流

```
用户说话
  ↓
录音（Mac麦克风）
  ↓
MiMo ASR → 文本
  ↓
Dify工作流（Qwen大脑）
  ├─ 意图识别
  ├─ 工具调用（天气/搜索）
  └─ 生成JSON：{speech, emotion, action}
  ↓
MiMo TTS → 语音（带情感）
  ↓
Linly-Talker（H20 DSW）
  └─ 数字人驱动 → 视频
  ↓
Mac播放（视频+音频）
```

## 显存分配

H20 96GB显存分配：
- LLM大脑：0GB（用API）
- ASR：0GB（用MiMo API）
- TTS：0GB（用MiMo API）
- 数字人驱动：~40GB（MuseTalk/SadTalker）
- 预留：~56GB（可用于模型优化/并发）

## 成本估算

### MiMo API
- ASR：约¥0.006/分钟
- TTS：约¥0.02/分钟

### Qwen API
- 输入：¥0.4/百万tokens
- 输出：¥1.2/百万tokens
- 免费额度：新用户100万tokens

### 预估（按1小时对话计算）
- ASR：60分钟 × ¥0.006 = ¥0.36
- TTS：~30分钟 × ¥0.02 = ¥0.6
- Qwen：~5万tokens × ¥0.8/百万 = ¥0.04
- **总计：~¥1/小时**

## 优势总结

1. **0本地显存** - 音视频API化，显存全给数字人驱动
2. **技术栈统一** - MiMo同时提供ASR+TTS，减少集成复杂度
3. **成本可控** - 约¥1/小时，课设demo完全够用
4. **情感表达丰富** - MiMo TTS支持8种音色+情感标签
5. **工具调用稳定** - Qwen原生支持Function Calling
